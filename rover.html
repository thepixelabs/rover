One stack. Three surfaces.

altergo runs it · Dispatch oversees it · Rover reaches it

Dispatch is the desktop cockpit for your AI coding agents — Claude, Codex, Gemini, Copilot — running in one place. altergo is the runtime underneath that keeps every account and session alive across restarts. Rover is the pocket terminal: SSH from your phone to your Mac and check, launch, or kill any agent in two keystrokes. One stack, three surfaces: altergo runs it, Dispatch oversees it, Rover reaches it.

// four agents. one surface.

One Dispatch. Every agent, on the same page.

One Dispatch install gives every AI coding agent on your machine — Claude Code, Gemini CLI, GitHub Copilot CLI, and OpenAI Codex CLI — a shared view of your project and a shared set of hands to move it forward.

One install wires all four CLIs.

The Settings panel writes the correct config for each provider automatically — Gemini’s httpUrl field, Copilot’s tools: ["*"], Codex’s TOML format. You do not touch config files.

11 shared MCP tools, identical across providers.

Any connected agent can call dispatch_get_state, dispatch_update_phase_status, dispatch_spawn_agent, and eight more. Gemini sees the same tools Claude does.

Single reactor keeps state consistent.

All agents read and write the same .tasks/ blackboard through Dispatch. Phase lifecycle is governed by one reactor, so concurrent updates from different providers do not produce conflicting state.

Install MCP tools

Copilot can’t be spawned by Dispatch — no headless CLI mode. Gemini and Copilot require re-installing when tokens rotate; the installer handles the update in one click. See per-provider guides for details.

// what rover does

Four chords, one phone, no laptop.

Three things Rover is for. Everything else is in the README.

Check on agents from the couch.

Lid's closed. Night shift is running. You pull the phone out, SSH in, hit D. Eighteen agents laid out in two columns. Everyone's fine. You put the phone down.

Unblock a stuck agent in two keystrokes.

Agent stopped on a permission prompt. You're in bed. Y to select it, Y again to yolo-resume. It's running. You didn't open a laptop.

Kill a runaway without opening the lid.

Something's spiraling. Token counter is climbing. You SSH in, X, confirm. Dead. The codebase fears you.

// voice input

Dictate. Agent moves.

Keyboard mic. Three sentences. Claude is already working.

Unblock with words, not a keyboard.

Agent hit a permission wall while you walk back from coffee. You open Rover, SSH in, dictate: “just trust the schema, accept the tool call, carry on.” It goes.

Kick off a plan hands-free.

Couch. An idea lands for the refactor you’ve been putting off. You dictate three sentences into the SSH session. Altergo spins it up. The first pass exists before you reach your desk.

Redirect mid-flight in plain English.

Agent is 40% through, going the wrong way. You say: “stop after the current file, add tests before moving on, prioritize the auth module.” It pivots. Nothing else needed.

You did not open your laptop.

// install

Paste one line.

Pick your flavor. Rover ships on Homebrew, pipx, uv, and a curl-to-shell fallback.

pipx or uv works on any Linux box with Python 3.11+. Homebrew is the macOS one-liner.

Rover runs on macOS and Linux. Windows isn't supported yet — try WSL.

// requirements

Three things. No substitutes.

Rover is a terminal client. It needs a terminal to reach.

host
macOS or Linux — Windows is WSL-only, not native yet
phone
any SSH client (Blink, Termius, a-Shell, browser terminal)*
agent stack
altergo — Rover drives altergo sessions, not raw provider CLIs

* Or just run rover on your laptop — same TUI, no SSH, when you're already at the desk.

Already running overnight agents and checking on them from bed. That is the use case. If the stack above is not complete, fix the shortest missing piece and come back.
